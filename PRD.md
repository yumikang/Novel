* 2차 창작(팬픽션) 지원 + 포스타입 특화
1. 제품 개요
NovelMind는 웹소설 및 2차 창작 작가를 위한 AI 기반 창작 도우미입니다. 기존 소설 텍스트를 분석하여 캐릭터 일관성, 복선 관리, 장르 패턴, 분위기 유지를 고려한 다음 전개 아이디어를 제안합니다.
특히 포스타입(Postype) 플랫폼에 특화된 UI/UX를 제공하며, 2차 창작 시 원작 캐릭터 일관성 유지를 핵심 기능으로 제공합니다.
1.1 목표 사용자
	•	포스타입에서 팬픽션/2차 창작을 연재하는 작가
	•	원작 캐릭터 OOC(Out of Character) 방지가 중요한 작가
	•	장기 연재로 복선 관리가 어려운 작가
	•	창작 슬럼프를 겪고 있는 작가
1.2 핵심 가치
	•	비용 제로: 프롬프트 생성 → ChatGPT/Claude에 붙여넣기 방식
	•	2차 창작 특화: 원작 캐릭터 DB + OOC 방지 시스템
	•	포스타입 최적화: 팬픽 문화에 맞는 UI/UX
	•	일관성: 캐릭터/복선/톤 추적으로 연재물 품질 유지
1.3 타겟 플랫폼: 포스타입
국내 2차 창작/팬픽션의 대표 플랫폼인 포스타입에 단일 특화합니다.
	•	주요 장르: BL, 팬픽션, 로맨스, 판타지
	•	특징: 시리즈 연재, 구독 시스템, 성인 인증
	•	UI 참고: 포스타입의 깔끔한 에디터 스타일 벤치마킹
2. 핵심 기능 (5대 기둥)
NovelMind의 차별화된 핵심 기능입니다. 2차 창작 지원이 추가되어 5대 기둥으로 확장되었습니다.

2.2 캐릭터 DB → 일관성 유지
문제: 장기 연재 시 캐릭터 성격, 말투, 설정이 흔들림
해결: 캐릭터별 프로필 DB 구축 및 프롬프트 자동 주입
저장 항목
	•	기본 정보: 이름, 나이, 외모, 직업
	•	성격 키워드: 냉철함, 츤데레, 유머러스 등
	•	말투 패턴: 예시 대사 3~5개
	•	관계도: 다른 캐릭터와의 관계
	•	변화 이력: 성장/변화 포인트 기록
원작 vs 오리지널 구분
	•	원작 캐릭터: 수정 불가 (AU 설정 적용만 가능)
	•	오리지널 캐릭터: 자유롭게 수정 가능

2.5 분위기/톤 유지 시스템
문제: AI 제안이 원작 분위기와 맞지 않아 이질감 발생
해결: 원문 분석 → 톤 프로필 생성 → 프롬프트에 반영
분석 항목
	•	문체: 간결체/만연체, 구어체/문어체, 1인칭/3인칭
	•	분위기: 어둡고 긴장감 있는 / 밝고 유쾌한 / 달달한
	•	템포: 빠른 전개 / 천천히 분위기 쌓기 / 슬로우번
	•	대사 비중: 대사 중심 / 서술 중심 / 균형형
	•	수위: 전연령 / 15세 / 성인 (포스타입 등급 연동)
3. 사용자 플로우
3.1 2차 창작 프로젝트 시작
	•	원작 선택 (프리셋 또는 직접 입력)
	•	타임라인/AU 설정 선택
	•	사용할 원작 캐릭터 선택
	•	오리지널 캐릭터 추가 (선택)
	•	장르, 수위 설정
	•	프로젝트 생성 완료
3.2 아이디어 생성
	•	최근 원고 붙여넣기 (마지막 1~2화)
	•	등장 캐릭터 자동 감지 + 수동 선택
	•	분석 옵션 선택 (캐릭터/복선/장르/톤)
	•	출력 AI 선택 (Claude/GPT/Gemini)
	•	[프롬프트 생성] 클릭
	•	[복사] → AI 채팅에 붙여넣기
	•	AI 응답 확인 및 활용
4. 기술 명세
4.1 기술 스택
	•	Frontend: Next.js 14+ (App Router), TypeScript, Tailwind CSS
	•	Storage: LocalStorage (Phase 1) → Supabase (Phase 2)
	•	UI Components: shadcn/ui (포스타입 스타일 커스텀)
	•	배포: Vercel
4.2 데이터 구조
OriginalWork (원작)
id, title, mediaType, canonCharacters[], worldRules[], timeline[], source
FanficProject (2차 창작 프로젝트)
id, originalWorkId, timelinePoint, auSettings[], rating, genre[]
Character (캐릭터)
id, projectId, name, isCanon, personality[], speechPatterns[], relationships[]
Foreshadow (복선)
id, projectId, title, description, status, plantedAt, expectedPayoff
ToneProfile (톤 프로필)
writingStyle, atmosphere, pacing, dialogueRatio, rating

6. 경쟁사 분석
6.1 뮤블(Muvel)
	•	강점: 플랫폼별 스타일 프리셋, 깔끔한 UI
	•	약점: 2차 창작 특화 기능 없음, AI 아이디어 제안 없음
	•	차별화: NovelMind는 원작 DB + OOC 방지에 특화
6.2 일반 AI 챗봇
	•	강점: 범용적 활용 가능
	•	약점: 매번 원작 설정 설명 필요, 캐릭터 OOC 빈번
	•	차별화: NovelMind는 원작 컨텍스트 자동 주입 + 캐릭터 일관성 보장
